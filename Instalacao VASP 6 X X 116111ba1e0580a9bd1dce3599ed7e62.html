Compilacao HDF5:

Configurar a compilação:

cd hdf5-1.X.X
./configure --enable-parallel --enable-fortran

→ Compilação:

make 
make check
make install
make check-install

Após a compilação bem-sucedida, adicionar o diretório bin e o lib ao path e LD_LIBRARY_PATH, respectivamente:

export PATH="<path-to-hdf5-bin>/bin:$PATH"
export LD_LIBRARY_PATH="<path-to-hdf5-lib>/lib:$LD_LIBRARY_PATH"

Makefile include com intel oneapi, hdf5:

# Default precompiler options
CPP_OPTIONS = -DHOST=\"LinuxGNU\" \
              -DMPI -DMPI_BLOCK=8000 -Duse_collective \
              -DscaLAPACK \
              -DCACHE_SIZE=4000 \
              -Davoidalloc \
              -Dvasp6 \
              -Duse_bse_te \
              -Dtbdyn \
              -Dfock_dblbuf \
              -D_OPENMP

CPP         = gcc -E -C -w $*$(FUFFIX) >$*$(SUFFIX) $(CPP_OPTIONS)

FC          = mpif90 -fopenmp
FCL         = mpif90 -fopenmp

FREE        = -ffree-form -ffree-line-length-none

FFLAGS      = -w -ffpe-summary=invalid,zero,overflow

OFLAG       = -O3
OFLAG_IN    = $(OFLAG)
DEBUG       = -O0

OBJECTS     = fftmpiw.o fftmpi_map.o fftw3d.o fft3dlib.o
OBJECTS_O1 += fftw3d.o fftmpi.o fftmpiw.o
OBJECTS_O2 += fft3dlib.o

# For what used to be vasp.5.lib
CPP_LIB     = $(CPP)
FC_LIB      = $(FC)
CC_LIB      = gcc
CFLAGS_LIB  = -O
FFLAGS_LIB  = -O1
FREE_LIB    = $(FREE)

OBJECTS_LIB = linpack_double.o

# For the parser library
CXX_PARS    = g++
LLIBS       = -lstdc++

##
## Customize as of this point! Of course you may change the preceding
## part of this file as well if you like, but it should rarely be
## necessary ...
##

# When compiling on the target machine itself, change this to the
# relevant target when cross-compiling for another architecture
FFLAGS     += -march=native

# For gcc-10 and higher (comment out for older versions)
#FFLAGS     += -fallow-argument-mismatch

# BLAS and LAPACK (mandatory)
#OPENBLAS_ROOT ?= /path/to/your/openblas/installation
#BLASPACK    = -L$(OPENBLAS_ROOT)/lib -lopenblas
BLASPACK    = -L/opt/intel/oneapi/mkl/2024.0/lib -lmkl_scalapack_lp64 -Wl,--no-as-needed -lmkl_gf_lp64 -lmkl_sequential -lmkl_core -lmkl_blacs_openmpi_lp64 -lpthread -lm -ldl 

# scaLAPACK (mandatory)
#SCALAPACK_ROOT ?= /path/to/your/scalapack/installation
SCALAPACK   = -L/opt/intel/oneapi/mkl/2024.0/lib -lmkl_scalapack_lp64 -Wl,--no-as-needed -lmkl_gf_lp64 -lmkl_sequential -lmkl_core -lmkl_blacs_openmpi_lp64 -lpthread -lm -ldl

LLIBS      += $(SCALAPACK) $(BLASPACK)

# FFTW (mandatory)
#FFTW_ROOT  ?= /path/to/your/fftw/installation
LLIBS      += -L/usr/lib/x86_64-linux-gnu/ -lfftw3 -lfftw3_omp
INCS       += -I/usr/local/include/

# HDF5-support (optional but strongly recommended)
# Nota: Versao 6.1 do VASP tem um bug com o uso do HDF5 em calculos de NEB!
# Para resolver bastar compilar o programa ignorando esse bloco:
CPP_OPTIONS += -DVASP_HDF5
LLIBS	    += -L/home/caique/softwares/hdf5-1.10.4/hdf5/lib/ -lhdf5_fortran
INCS        += -I/home/caique/softwares/hdf5-1.10.4/hdf5/include/

Depois de acertar o makefile.include:

make DEPS=1 -j N

Compilação VASP Com Vasp Transition State Tools (VTST):

Código do grupo de Henckelmann para estudo de estados de transição incluído como módulo externo do VASP.

→ Página do grupo de Henckelmann:

→ Página de Download do VTST (VASP):

→ Instalação (segundo instruções da página do VASP):

  1. Fazer o download dos arquivos do VTST:
wget https://theory.cm.utexas.edu/code/vtstcode-199.tgz
  1. Para versões do VTST mais recentes do que 2.04, é preciso modificar o arquivo main.F (vasp/src) do VASP para cálculos de NEB:
  • Substituir:
CALL CHAIN_FORCE(T_INFO%NIONS,DYN%POSION,TOTEN,TIFOR, &
     LATT_CUR%A,LATT_CUR%B,IO%IU6)

por:

CALL CHAIN_FORCE(T_INFO%NIONS,DYN%POSION,TOTEN,TIFOR, &
     TSIF,LATT_CUR%A,LATT_CUR%B,IO%IU6)

Para o VASP 6.2 e versões mais recentes, modificar:

IF (LCHAIN) CALL chain_init( T_INFO, IO)

por 

CALL chain_init( T_INFO, IO)

*Para o VASP 6.2.0, utilizar a versão vtstcode-199/6.2.

  1. Copiar todos os arquivos do diretório do vtstcode para o diretório src do VASP. ATENÇÃO! O arquivo chain.F é substituido. Convém manter uma cópia do arquivo original.
  1. Modificar o arquivo .objects (vasp/src/.objects) e adicionar os seguintes objetos na variável SOURCE antes do chain.o:
    !In vasp/src/.objects:
...
bfgs.o dynmat.o instanton.o lbfgs.o sd.o cg.o dimer.o bbm.o \
fire.o lanczos.o neb.o qm.o opt.o \
chain.o \
  1. Para compilação em paralelo, adicionar libs às dependências do arquivo makefile do diretório src:
    !IN vasp/src/makefile:
...
dependencies: sources libs

Após esses passos, o vasp pode ser normalmente compilado:

make DEPS=1 -jN

Instalação do VASPSol

→ VASPSol: código mais antigo. Arquivos de patch estão disponíveis diretamente no github.

  1. Clonar o repositório do Github:
git clone https://github.com/henniggroup/VASPsol.git
  1. Copiaro arquivo solvation.F para o diretório src do VASP;
  1. Modificar o arquivo .objects no diretório src do VASP de modo que o objeto solvation.o apareça antes de pot.o.
  1. Modificar o arquivo makefile.include :
CPP_OPTIONS += -Dsol_compat
  1. Aplicar o patch utilizando o arquivo vaspsol_fermi_vasp6.patch:
patch -p0 < <path-to-patch-file>

Compilar o VASP normalmente:

make DEPS=1 -jN
